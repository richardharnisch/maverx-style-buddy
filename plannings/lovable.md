
# Maverx Training Builder — Agentic Skill (Tier 3)

A self-contained skill that any agent runtime (Codex, Claude Code, Lovable) can load to take a trainer from one sentence → full certification track (e.g. Lean Black Belt, 8 sessions × 2h, DMAIC) as editable .pptx decks plus pre/post-bites, case handouts, and a track overview.

The skill is the product. The agent host is the UI. No web app, no server.

---

## 1. What we learned from the supplied style guide

The uploaded `Maverx - Presentation Style Guide for Hackaton.pptx` is both the **style guide** and the **master template** — it already contains every layout we need, grouped into 5 sections matching the didactic model:

1. Pre-training slides (cover, About, Script, Title, Timetable, Debrief & Next Steps)
2. Theory / topic slides (with several layout variants)
3. Exercise slides (Exercise + Debrief pair)
4. Wrap-up & Next Steps slides (KPI ladder, Resources, "Share it" close)
5. Other / Asset templates (1–4) + Break + Debrief assets

Locked house-style rules to encode in `references/house_style.md`:

- **Fonts:** Space Grotesk Bold 33pt (titles), Space Grotesk 22pt (subtitle), Raleway or Space Grotesk 15+pt (body); bullets only for sub-points (20+pt regular, 18pt for dense slides); indent via Tab.
- **Color palette (exact hex):**
  - Primary Dark `#0D006A`, Dark Purple `#1A0040`, Deep Purple `#3F0576`
  - Accents: Rose Red `#EF4453`, Orange `#F48A28`, Teal `#00B0F0`
  - Tints: `#BCB3FF`, `#F7B8C0`, `#FAD0A8`, `#9FE6FF`; Dark Grey `#262626`
  - Backgrounds: Off-white `#F2F2F2`, BG Lavender `#EDE9FF`, Rose `#FEF0F1`, Orange `#FDEBDB`, Teal `#E7F9FF`
- **Color rule:** lighter background → darker highlights; max 3 base colors per slide; new colors only via brighten/darken existing ones.
- **Copy rules:** avoid `/` and `,` where possible; "left or right" / "left & right" instead of "left/right"; never use `•` on the first line.
- **Layout discipline:** don't move title/subtitle without copying placement to all other slides; respect Safe / Flexible / No-go margin zones; footer is `maverx.nl` + social icons on every content slide.
- **Section dividers** use the Pre-training / Theory / Exercise / Wrap-up cover styles already present in the master.

This file becomes the canonical `assets/maverx_master.pptx`; `scripts/inspect_master.py` enumerates its layouts and writes `assets/master_index.json` so the builder fills placeholders by index rather than redrawing shapes.

## 3. Intake (hard gate)

Driven by `system_intake.md` + `intake.schema.json`. Required questions (brief):

1. Topic / skill
2. Target audience
3. Knowledge level
4. Total duration (Tier 3 default: 8 × 2h)
5. Primary learning objective

Tier-3 follow-ups: certification name, fictional company/sector for the running case, tools/data realism, language (NL/EN), assessment style, branding overrides (logo swap, alternate palette).

Rule: `run_tier3.py` refuses to proceed until `intake.schema.json` validates **and** an LLM self-check pass returns `completeness_score ≥ threshold` with no `vague_fields`. Vague answers trigger targeted follow-ups, never silent guesses.

## 4. Generation pipeline (Tier 3)

```
intake.json
   │
   ▼
plan_track.py ── track_plan.json
   • DMAIC (or chosen backbone) over 8 sessions
   • fictional company + persistent business case + dataset spec
   • per-session objectives, dependencies, post_bite(N)↔pre_bite(N+1) handshake
   │
   ▼  for N = 1..8
plan_session.py ── sessions/N/session_plan.json
   • slide-by-slide blueprint mapped to didactic blocks
   • each slide pinned to a master layout_id from master_index.json
   • timing budget summing to session length
   • exercise spec (tool-step OR case-handout reference)
   • Mentimeter recap slide for sessions ≥ 2
   • speaker notes draft (all 5 fields)
   │
   ├── build_deck.py        → sessions/N/session_N.pptx
   ├── build_bites.py       → sessions/N/pre_bite.docx, post_bite.docx
   └── build_case_handout.py→ sessions/N/case_handout.docx
   │
   ▼
build_track_overview.py ── track_overview.docx
   │
   ▼
qa_deck.py (per deck + track-level)
   • LibreOffice → PDF → pdftoppm; visual + textual checks
   • opens cleanly, all text editable, fonts/colors match house_style.md,
     every slide has all 5 notes fields, timing sums correct,
     post_bite N ↔ pre_bite N+1 narrative continuity
   • emits qa_report.md; orchestrator re-prompts LLM to fix flagged slides
```

The orchestrator loops QA → fix → re-QA per deck, then surfaces results to the user; the user can edit `track_plan.json` or any `session_plan.json` between stages (human-in-the-loop, as the rubric demands).

## 5. PowerPoint authoring rules (House Style Compliance — 20 %)

- `inspect_master.py` is run once; results cached in `assets/master_index.json` (layout name → idx, placeholder roles, theme color slots, master fonts).
- `build_deck.py` does `slide_layout = prs.slide_layouts[idx]` and fills placeholders by index — **never** redraws shapes, **never** inserts images of text.
- Tables are real `python-pptx` tables (editable).
- Logo, footer (`maverx.nl` + social icons), and palette are sourced from `house_style.md`.
- Swapping the style guide = drop a new master into `assets/maverx_master.pptx`, rerun `inspect_master.py`, regenerate `house_style.md` (documented in README).

---

## 6. Speaker notes contract

Every slide's notes are a fixed Markdown block parsed/validated by `qa_deck.py`:

```
**Aim:** …
**Time:** … min
**Instructions:**
1. …
**Reflective question:** …
**Debrief:** …
```

Missing field → QA failure → regenerate that slide only.

---

## 7. OpenRouter usage

- Single client `scripts/openrouter_call.py` reads `OPENROUTER_API_KEY` (provided key documented as dev default in README; users override via env).
- Default model mapping in `SKILL.md`:
  - Planning stages (track + session): strong reasoning model (e.g. `anthropic/claude-sonnet-4.5` or `openai/gpt-5.1`).
  - Bulk slide/bite writing: cheaper fast model.
- All calls go through one helper for retries, JSON-mode, and token accounting. Tier-3 token load is mitigated by per-session planning + streaming writes to disk (no monolithic context).

---

## 8. Host integration (agent runtime behaviour)

1. Detect trigger ("build a Maverx training", "Tier 3 certification", …).
2. Read `SKILL.md` → run intake conversation → persist `intake.json`.
3. Call `python scripts/run_tier3.py --intake intake.json --out ./out/<slug>`.
4. Stream progress; surface `qa_report.md`; offer the user the chance to edit `track_plan.json` / any `session_plan.json` between stages.
5. Deliver the `out/` folder as the final bundle.

## 10. Open items / assumptions

- The supplied `.pptx` is treated as the canonical master; if a separate Style Guide PDF or example .docx appears later, drop them in `assets/` and add their distilled rules to `house_style.md` / `references/didactic_model.md`.
- Rendering QA requires LibreOffice + Poppler in the host sandbox (already present in Lovable/Codex; documented for Claude Code).
- OpenRouter key shipped as a dev default only; production users set `OPENROUTER_API_KEY` themselves.
- Mentimeter recap = generated recap slide + companion `mentimeter_questions.md` (no live API in v1).
- Tier 1 / Tier 2 fall out of the same pipeline by swapping the orchestrator preset — kept out of v1 scope to keep Tier 3 solid.
